# v0.1.0

First public MVP release of Skill Health Dashboard.

## Highlights

- Added local skill inventory scanning via `skillcheck scan skills`.
- Added optional Codex local history importer via `skillcheck import codex`.
- Added one-command pipeline via `skillcheck refresh` (`scan` + `import` + `aggregate` + `doctor`).
- Added structured summary output via `skillcheck summary --format json` (default now lists all skills).
- Added doctor diagnostics via `skillcheck doctor` for support and issue reporting.
- Replaced legacy status scoring with V2 six-dimension scoring:
  - `security_score`
  - `clarity_score`
  - `overlap_score`
  - `stability_score`
  - `efficiency_score`
  - `confidence_score`
- Added V2 status classes: `Qualified`, `Watch`, `Unqualified`.
- Added local-only privacy clarifications and importer constraints in docs.

## User guide (recommended flow)

```bash
python -m pip install -e .
skillcheck init
skillcheck refresh
skillcheck summary
skillcheck dashboard
```

For machine-readable terminal output:

```bash
skillcheck summary --format json
```

For inventory + import + aggregate step-by-step:

```bash
skillcheck scan skills
skillcheck import codex
skillcheck aggregate
skillcheck doctor
```

## Privacy notes

- Data is stored locally in SQLite by default.
- No session log upload is performed.
- Prompt/response full text is not stored by the importer.
- `raw_event_ref` is a local path + line reference for troubleshooting only.

## Known scope limits

- Dashboard is currently centered on Overview and API payloads.
- Importer events are currently proxy-based (`SKILL.md` load events), with direct platform events planned later.
