# v0.1.0

First public MVP release of Skill Health Dashboard.

## Highlights

- Added local skill inventory scanning via `skill-health scan skills`.
- Added optional Codex local history importer via `skill-health import codex`.
- Added one-command pipeline via `skill-health refresh` (`scan` + `import` + `aggregate` + `doctor`).
- Added structured summary output via `skill-health summary --format json` (default now lists all skills).
- Added doctor diagnostics via `skill-health doctor` for support and issue reporting.
- Replaced legacy status scoring with V2 six-dimension scoring:
  - `security_score`
  - `clarity_score`
  - `overlap_score`
  - `stability_score`
  - `efficiency_score`
  - `confidence_score`
- Added V2 status classes: `Qualified`, `Watch`, `Unqualified`.
- Added local-only privacy clarifications and importer constraints in docs.

## User guide (recommended flow)

```bash
python -m pip install -e .
skill-health init
skill-health refresh
skill-health summary
skill-health dashboard
```

For machine-readable terminal output:

```bash
skill-health summary --format json
```

For inventory + import + aggregate step-by-step:

```bash
skill-health scan skills
skill-health import codex
skill-health aggregate
skill-health doctor
```

## Privacy notes

- Data is stored locally in SQLite by default.
- No session log upload is performed.
- Prompt/response full text is not stored by the importer.
- `raw_event_ref` is a local path + line reference for troubleshooting only.

## Known scope limits

- Dashboard is currently centered on Overview and API payloads.
- Importer events are currently proxy-based (`SKILL.md` load events), with direct platform events planned later.
